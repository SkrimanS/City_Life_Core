# SDK ZIP Package

Version: **1.0.0**

The SDK ZIP package is an install-layout archive produced by CPack.

It is intended to be unpacked and consumed through CMake with `find_package(CityLifeCore CONFIG REQUIRED)`. It also carries documentation and examples, including C ABI and initial C# / Unity integration examples.

## Build ZIP package

```bash
cmake -S . -B build-sdk-zip -DCLC_BUILD_TESTS=OFF -DCLC_BUILD_EXAMPLES=OFF -DCLC_BUILD_TOOLS=OFF
cmake --build build-sdk-zip --config Release
cpack --config build-sdk-zip/CPackConfig.cmake -G ZIP
cmake -E sha256sum city-life-core-sdk-*.zip > SHA256SUMS.txt
```

## Consume ZIP package

Unpack the archive and point `CMAKE_PREFIX_PATH` at the unpacked install prefix. Consumers can then use:

```cmake
find_package(CityLifeCore CONFIG REQUIRED)
target_link_libraries(my_app PRIVATE CityLifeCore::core)
```

## Expected contents

The ZIP package should contain the installed SDK layout:

- public headers;
- native library artifacts generated by the selected build configuration;
- CMake package config files;
- documentation;
- examples;
- example data packs.

Integration examples should include:

```text
examples/c_abi_consumer/
examples/csharp_unity/
```

The C# / Unity files are source examples and wrappers. Unity still needs the native shared library built for the target platform and copied into `Assets/Plugins`.

## Native plug-in note

For Unity or other native plug-in scenarios, prefer producing a shared-library SDK archive:

```bash
cmake -S . -B build-sdk-zip-shared -DBUILD_SHARED_LIBS=ON -DCLC_BUILD_TESTS=OFF -DCLC_BUILD_EXAMPLES=OFF -DCLC_BUILD_TOOLS=OFF
cmake --build build-sdk-zip-shared --config Release
cpack --config build-sdk-zip-shared/CPackConfig.cmake -G ZIP
```
